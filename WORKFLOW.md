# Decisiones de Workflow - Proyecto Fase II
## ¿Cuándo usamos rebase?

- Para combinar 5 commits de documentación del proyecto
- Para combinar 3 commits de relacionados con el código del proyecto
- Para combinar 2 commits relacionados con la estructura del proyecto

## ¿Cuándo usamos merge?

- En este caso no usamos merge, pero sería en casos donde se desea tener todo el historial de cambios del proyecto

## ¿Cuándo usamos cherry-pick?

- Se usó para llevar la el codigo de la función B a la rama principal

## Lecciones aprendidas

- El rebase es util porque ayuda a mantener el historial de commits más organizado pero se debe tener mucho cuidado al subir los cambios usándolo.
